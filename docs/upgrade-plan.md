# triage-copilot 升级说明

## 当前阶段目标

项目当前已经从“LLM 直接猜挂哪个科”的原型，升级为结构化分诊底座，主链路稳定落在以下骨架上：

1. 用户画像标准化
2. 路径标签识别
3. 疾病候选识别
4. 疾病到医学能力映射
5. 医学能力到本地科室映射
6. 医生扩展推荐
7. AI 解释生成

## 已完成能力

### 第一阶段

- 新增 `com.example.triage` 分层结构
- 落地疾病主表、能力目录、本地医院和科室映射表
- 实现 `POST /api/triage/assess`
- 实现 `POST /api/base-data/import` 和 `GET /api/base-data/check`

### 第二阶段

- 引入复杂路径标签
- 支持儿童发热、老年记忆下降、男性排尿异常、脊柱疼痛、移植术后随访
- 医学能力目录支持标准科室、亚专科、特殊路径能力

### 第三阶段

- 增加医生推荐扩展位
- 响应结构新增 `doctorRecommendations`

### 第四阶段

- 增加 AI 辅助疾病候选补召回
- AI 只允许在已入库白名单中补充 disease code

### 第五到第七阶段

- 增加模板接口
- 增加待人工复核查询与关闭
- 增加导入任务追踪和失败明细

## 本轮修正

本轮主要完成了以下修正：

- 将结构示例医院从“上海 demo”改为中性示例数据
- 统一年龄分层边界：`child 0-11`、`adolescent 12-17`、`adult 18-64`、`elderly 65+`
- 修复能力层级缺口，为 `cap_spine_surgery` 补齐父级 `cap_orthopedics`
- 增加本地科室到医学能力的自动映射规则，避免导入后全部停留在人工映射
- 增加武汉导入适配层，支持用武汉真实字段名导入疾病和本地科室
- 为 AI 疾病补召回增加审计日志和高风险症状保护
- 重写 README 和核心文档，使接口说明、初始化方式与代码一致

## 当前导入策略

### 支持的数据集类型

- `disease`
- `department`
- `wuhan_disease`
- `wuhan_department`

### 当前导入流程

1. 读取 CSV/XLSX
2. 适配字段
3. 清洗别名和症状关键词
4. 入库基础表
5. 自动尝试医学能力映射
6. 生成失败日志和待人工复核项
7. 记录导入任务结果

## 已知限制

- AI 补召回仍是轻量辅助版，还没有引入更细的置信度阈值和人工审批流
- 武汉导入适配目前是首版字段别名兼容，后续仍需要根据真实源文件继续扩充
- 医学能力自动映射目前基于规则，不等于最终人工确认结果
- `crowd_constraint` 仍主要作为关系层语义保留，后续可进一步下沉到关系级约束执行

## 下一步建议

- 增加 AI 审计日志查询接口
- 增加人工确认自动映射结果的接口
- 接入武汉真实疾病表和真实医院科室清单做一次全量试导
- 为高风险场景增加专门的急诊/立即就医提示层
